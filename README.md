# 任务清单 (Task Tracker)

一个用于跨会话管理长期项目任务的 Claude Code Skill。

## 📖 简介

本 Skill 通过在项目根目录维护一个 `instruction.md` 文件，实现对跨多个对话会话的项目工作进行系统化跟踪管理。

`instruction.md` 不是普通的笔记，它是项目的**权威执行账本**，包含：
- 项目目标与范围边界
- 当前阶段与步骤
- 阻塞问题
- 验证记录
- 变更文件与行号范围
- 唯一下一步执行入口

## 🚀 使用场景

当您需要：
- 继续之前开始的任务（`继续任务 X`）
- 查询任务进度（`X 任务进行到哪里了`）
- 启动需要跨会话跟踪的新项目
- 从已有的 `instruction.md` 继续工作
- 需要严格的任务清单而非松散的计划

## 📋 核心特性

### 状态标记系统
- 🟢 `[✔]` 完成
- 🟡 `[!]` 遇到问题/需注意
- 🔴 `[❌]` 错误/阻塞
- ⚪ `[ ]` 未开始

### 严格的层级执行顺序
- 只有叶子节点可直接执行
- 父节点状态由子节点推导
- 校验点 `[校验点]` 必须在进入下一阶段前通过

### 变更追踪
每个完成的步骤必须记录：
- 变更文件路径
- 行号范围
- 变更类型（新增/修改/删除）

### Git 提交边界
- 一个完成的步骤对应一个 commit
- Commit 应包含该步骤相关的文件
- 任务记录更新应包含在同一次 commit 中

## 📝 示例

```markdown
## 📋 进度清单
*状态图例：🟢 [✔] 完成 | 🟡 [!] 遇到问题/需注意 | 🔴 [❌] 错误/阻塞 | ⚪ [ ] 未开始*

### 阶段 1：项目初始化
- ⚪ [ ] 步骤 1.1：需求分析
  > **目标：** 明确项目需求与边界
  > **执行动作：** 分析用户需求文档
  > **验证方式：** 输出需求清单
  > **完成标准：** 需求清单经确认
```
